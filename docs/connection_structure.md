# 연결 구조 설명

## 1. 우수 흐름

폭우가 내리면 빗물은 먼저 빗물받이로 들어갑니다.

```text
폭우
→ 빗물받이 CB1~CB5
→ 우수 연결관
→ 우수관거 본관
→ 빗물펌프장 유입부
→ 빗물펌프
→ 펌프 토출관
→ 하천 방류구
→ 하천
```

현재 모델에서는 `CB3`와 `CB3_CONN`이 일부러 취약하게 설정되어 있습니다.

즉 기본 폭우 시나리오에서 가장 먼저 문제가 생기는 곳은 전체 본관이 아니라 `CB3` 빗물받이와 그 연결관입니다.

## 2. 오수/합류식 흐름

생활오수는 생활오수 연결관을 통해 합류식 하수관거로 들어갑니다.

```text
생활오수
→ 생활오수 연결관 SAN_CONN
→ 합류식 하수관거 COMB_MAIN_1
→ 합류 맨홀 COMB2
```

`COMB2`는 중요한 분기점입니다.

정상 상황에서는 물이 차집관거를 통해 물재생센터로 갑니다.

```text
COMB2
→ 차집관거 INTERCEPTOR
→ 물재생센터 WWTP_STORAGE
→ 처리수 방류관 TREATED_OUT
→ 방류
```

## 3. 폭우 시 월류 흐름

합류식 구간에서 물이 너무 많아지면 일부 물은 물재생센터로 가지 않고 월류시설을 통해 하천으로 나갈 수 있습니다.

```text
COMB2
→ 우수토실/월류시설 CSO_WEIR
→ 월류 방류관 OVF_PIPE
→ 하천 방류구 RIVER_OVF_OUT
→ 하천
```

현재 시나리오에서는 `CSO_WEIR`가 크게 작동하지 않습니다.

월류시설 작동을 강조하려면 다음 중 하나를 추가하면 됩니다.

- 합류식 강우 유입량 증가
- 차집관거 용량 감소
- 물재생센터 처리 용량 감소
- 하천 수위 상승

## 4. 하천 역류 흐름

하천 수위가 높으면 방류구 쪽에서 물이 빠지지 못하거나, 반대로 관 안쪽으로 밀고 들어올 수 있습니다.

현재 하천 역류 시나리오에서는 다음 위치에서 역류 후보가 잡힙니다.

```text
OVF_PIPE
PUMP_FORCE_OUT
STORM_TO_PUMP
```

이건 실제 시스템에서 다음을 확인해야 한다는 뜻입니다.

- 하천 방류구 수위
- 역류 방지관 상태
- 펌프 토출관 상태
- 펌프장 유입부 수위

## 5. 센서 관점에서 보는 핵심 지점

| 지점 | 필요한 데이터 | 판단 목적 |
|---|---|---|
| 빗물받이 | 내부 수위 | 도로 빗물이 빠지고 있는지 확인 |
| 빗물받이 연결관 | 유량, 유속, 만관 여부 | 연결관 막힘 판단 |
| 우수 본관 상류/하류 | 수위, 유속 | 본관 병목 위치 추정 |
| 합류 맨홀 | 수위 | 차집/월류 분기 상태 확인 |
| 펌프장 유입부 | 수위, 유입량 | 펌프 가동 필요성 판단 |
| 펌프 토출관 | 유량, 역방향 유량 | 펌프가 실제 배수하는지 확인 |
| 월류 방류관 | 유량, 역방향 유량 | 월류 발생 및 하천 역류 확인 |
| 하천 방류구 | 하천 수위 | 방류 가능 여부 확인 |
