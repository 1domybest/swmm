# Windows SWMM GUI Import Guide

## 1. 어떤 파일을 열면 되나?

Windows EPA SWMM GUI에서 아래 파일을 열면 됩니다.

| 파일 | 의미 |
|---|---|
| `models/seoul_pilot_base.inp` | 기본 폭우 시나리오 |
| `models/seoul_pilot_backflow.inp` | 하천 수위 상승/역류 시나리오 |

## 2. 실행 순서

1. EPA SWMM을 실행합니다.
2. `File > Open`을 선택합니다.
3. `models/seoul_pilot_base.inp`를 엽니다.
4. 화면에 관망도가 나타나는지 확인합니다.
5. 실행 버튼을 눌러 시뮬레이션을 실행합니다.
6. 실행이 끝나면 `Report`에서 결과를 확인합니다.
7. 같은 방식으로 `models/seoul_pilot_backflow.inp`도 열어 비교합니다.

## 3. import하면 어떻게 보이나?

이 모델은 `.inp` 내부에 `[COORDINATES]`, `[VERTICES]`, `[MAP]` 정보가 들어 있습니다.

따라서 SWMM GUI에서 열면 다음 정도의 이미지화가 됩니다.

```text
노드: 빗물받이, 맨홀, 펌프장 유입부, 물재생센터, 방류구
관로: 우수 연결관, 우수 본관, 차집관거, 월류 방류관, 펌프 토출관
시설: 펌프, 우수토실/월류시설
```

하지만 이건 실제 사진 같은 그림이 아니라 SWMM의 2D 관망도입니다.

## 4. SWMM GUI에서 볼 수 있는 결과

실행 후에는 다음을 확인할 수 있습니다.

- 특정 맨홀의 수위 변화
- 특정 관의 유량 변화
- 특정 관의 유속 변화
- 침수 발생 노드
- 펌프 작동 시점
- 방류구 쪽 수위 영향
- 관로 프로파일 그래프

## 5. 발표용으로 더 보기 좋은 화면

SWMM GUI는 수리해석용 화면이라 처음 보면 직관성이 떨어질 수 있습니다.

발표나 설명용으로는 아래 파일을 브라우저로 여는 게 더 좋습니다.

```text
viewer/urban_flood_network_viewer.html
```

이 파일은 Chrome 또는 Edge에서 그냥 열 수 있습니다.

여기서는 다음을 볼 수 있습니다.

- 기본 폭우와 하천 역류 시나리오 전환
- 빗물받이, 본관, 펌프장, 월류시설, 물재생센터의 흐름
- 침수/병목 지점 색상 표시
- 역류 후보 지점 표시
- 각 지점 클릭 시 가상 센서값 요약

## 6. 주의할 점

현재 모델은 서울 전체 실측 GIS 모델이 아니라 과제 설명용 파일럿 모델입니다.

따라서 다음처럼 이해하면 됩니다.

```text
실제 서울 전역 모델
X

서울 배수 시스템의 핵심 시설과 문제 상황을 보여주는 축소 시뮬레이션
O
```

이 모델의 목적은 실제 센서 설치 전, SWMM 결과를 가상 센서값처럼 받아서 침수/역류/병목 판단 로직을 만드는 것입니다.
